## What it does for you

Layout Builder assembles frontend layouts from reusable widgets. A widget can hold text, media, actions, presentation settings, and translations; layouts decide where those widgets are used for a site and theme.

## Your screens

- **Websites -> Widgets**: create and edit reusable widget records, then see the layouts that use each widget.
- **Websites -> Layouts**: manage the layouts that connect page records to frontend rendering and update their widget assignments.
- **Websites -> Layout Presets**: review the available preset layouts, their mode, category, revision, and usage count.

## What you can do

- Create a widget from the configured widget type, then edit its content, media/assets, display settings, and translations.
- Reuse a widget in more than one layout, or use the widget and layout tools to move or replace widgets across layouts.
- Replicate, disable, delete, restore, and filter widgets by type, layout, language, status, or trash state.
- Review layout-preset usage before applying the appropriate layout pattern elsewhere in the admin.

## Where to find it

Open **Websites -> Widgets** for the reusable building blocks, then use **Layouts** to manage where those blocks render. The Widgets list also links to the layouts screen and includes a layouts action.

## Good to know

- A widget is not necessarily visible just because it exists: its status and optional visible-from/visible-until dates affect availability.
- Widgets and layouts are site/theme-aware, so check the record's context before reusing or bulk-changing it.
- Layout Presets are a catalogue/review screen; the current resource is list-only, rather than a full preset editor.
