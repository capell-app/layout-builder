# Theme Studio, Mosaic, and Capell Assistant Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Theme Studio Premium real in public rendering, then expose shallow creator-friendly Mosaic and Theme Studio adapters for the commercial Capell Assistant package.

**Architecture:** Theme Studio remains the premium theme/runtime layer, Mosaic remains the structured layout builder, and Capell Assistant becomes one commercial orchestration package. Integration uses shallow adapter-style module interfaces: Assistant owns capability registration and optional package wrappers, while package-owned Actions and Data objects stay in their original packages.

**Tech Stack:** PHP 8.2, Laravel, Filament, Livewire, Spatie Laravel Data, Lorisleiva Actions, Pest, Capell frontend render hooks, optional Capell Workspaces.

---

## Product Decisions

- The product name is **Capell Assistant**.
- Use **Capell Assistant** for all product-facing assistant naming.
- Theme Studio Premium must render real public pages before Assistant automation is treated as implementation-ready.
- Mosaic stays separate from Theme Studio. Mosaic composes structured layouts; Theme Studio translates portable content into premium visual systems.
- Default Theme remains the free infrastructure layer.
- Assistant uses shallow adapters and package-owned Actions. It must not reach into package internals.
- Foundation packages must not import Assistant. Optional AI wrappers live in the commercial Assistant package and activate only when the target package is installed.
- Assistant and Mosaic must prefer reuse before creation: existing themes, presets, patterns, widgets, assets, tags, layouts, and content are searched first.
- New PHP, Blade, schema configurator, or layout configurator code generated by Assistant must be developer-approved.
- Automated creation is draft/proposal-first in v1.

## Shallow Adapter Rule

Keep package integration deliberately small:

```php
<?php

declare(strict_types=1);

namespace Capell\Assistant\Contracts;

interface AssistantModule
{
    public function key(): string;

    public function label(): string;

    /**
     * @return array<int, \Capell\Assistant\Data\AssistantCapabilityData>
     */
    public function capabilities(): array;
}
```

The Assistant asks:

1. Which modules are available?
2. Which capabilities do they expose?
3. Which Action should run for this capability?

Each package decides how its own Action works. The Assistant does not learn Blog tags, SEO meta internals, or Theme Studio runtime details. For Mosaic, Assistant wraps the public layout-planning Actions while Mosaic remains free of commercial package imports.

## Implementation Status Update

As of May 2, 2026:

- `commercial` is an approved premium bundle and `capell-app/assistant` is documented as **Capell Commercial**.
- Theme Studio Admin has a first-party Workspaces publisher adapter. If Workspaces is installed, staged theme drafts are submitted for approval and the linked approved workspace activates the draft; otherwise standalone publish remains the fallback.
- Mosaic no longer imports Assistant classes. The Mosaic AI capability now lives in `capell-app/assistant` as an optional integration wrapper around Mosaic Actions.
- `CapellFrontendThemePageAdapter` maps the active frontend page and loaded Mosaic layout widgets into portable Theme Studio sections, including page-level widget assets.
- Capell package docs and the Capell app package catalogue have been updated for the new package names and bundle split.

namespace Capell\ThemeStudio\Core\Adapters;

use Capell\Frontend\Facades\Frontend;
use Capell\ThemeStudio\Core\Contracts\ThemePageAdapter;

<?php

declare(strict_types=1);

namespace Capell\ThemeStudio\Core\Assets;

use Capell\ThemeStudio\Core\Data\BrandProfileData;
use Illuminate\Support\Facades\File;

class ThemeTokenStore
{
    public function __construct(private readonly ?string $directory = null) {}

    public function put(string $themeKey, string $presetKey, BrandProfileData $brand): string
    {

<?php

declare(strict_types=1);

namespace Capell\Assistant\Actions;

use Capell\Assistant\Data\AssistantRunData;
use Capell\Assistant\Support\AssistantModuleRegistry;
use InvalidArgumentException;
use Lorisleiva\Actions\Concerns\AsObject;

class RunAssistantCapabilityAction
{
    use AsObject;

    public function handle(string $capabilityKey, array $input = []): AssistantRunData
    {
        $capability = app(AssistantModuleRegistry::class)->findCapability($capabilityKey);

        throw_unless($capability instanceof \Capell\Assistant\Data\AssistantCapabilityData, InvalidArgumentException::class, 'Assistant capability is not registered: ' . $capabilityKey);

        $output = $capability->action::run($input);

        return new AssistantRunData($capabilityKey, $output);
    }
}
