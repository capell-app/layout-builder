# Theme Studio, LayoutBuilder, and Capell AIOrchestrator Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Theme Studio Premium real in public rendering, then expose shallow creator-friendly LayoutBuilder and Theme Studio adapters for the commercial Capell AIOrchestrator package.

**Architecture:** Theme Studio remains the premium theme/runtime layer, LayoutBuilder remains the structured layout builder, and Capell AIOrchestrator becomes one commercial orchestration package. Integration uses shallow adapter-style module interfaces: AIOrchestrator owns capability registration and optional package wrappers, while package-owned Actions and Data objects stay in their original packages.

**Tech Stack:** PHP 8.2, Laravel, Filament, Livewire, Spatie Laravel Data, Lorisleiva Actions, Pest, Capell frontend render hooks, optional Capell PublishingStudio.

---

## Product Decisions

- The product name is **Capell AIOrchestrator**.
- Use **Capell AIOrchestrator** for all product-facing ai-orchestrator naming.
- Theme Studio Premium must render real public pages before AIOrchestrator automation is treated as implementation-ready.
- LayoutBuilder stays separate from Theme Studio. LayoutBuilder composes structured layouts; Theme Studio translates portable content into premium visual systems.
- Foundation Theme remains the free infrastructure layer.
- AIOrchestrator uses shallow adapters and package-owned Actions. It must not reach into package internals.
- Foundation packages must not import AIOrchestrator. Optional AI wrappers live in the commercial AIOrchestrator package and activate only when the target package is installed.
- AIOrchestrator and LayoutBuilder must prefer reuse before creation: existing themes, presets, patterns, widgets, assets, tags, layouts, and content are searched first.
- New PHP, Blade, schema configurator, or layout configurator code generated by AIOrchestrator must be developer-approved.
- Automated creation is draft/proposal-first in v1.

## Shallow Adapter Rule

Keep package integration deliberately small:

```php
<?php

declare(strict_types=1);

namespace Capell\AIOrchestrator\Contracts;

interface AIOrchestratorModule
{
    public function key(): string;

    public function label(): string;

    /**
     * @return array<int, \Capell\AIOrchestrator\Data\AIOrchestratorCapabilityData>
     */
    public function capabilities(): array;
}
```

The AIOrchestrator asks:

1. Which modules are available?
2. Which capabilities do they expose?
3. Which Action should run for this capability?

Each package decides how its own Action works. The AIOrchestrator does not learn Blog tags, SEO meta internals, or Theme Studio runtime details. For LayoutBuilder, AIOrchestrator wraps the public layout-planning Actions while LayoutBuilder remains free of commercial package imports.

## Implementation Status Update

As of May 2, 2026:

- `commercial` is an approved premium bundle and `capell-app/ai-orchestrator` is documented as **Capell Commercial**.
- Theme Studio Admin has a first-party PublishingStudio publisher adapter. If PublishingStudio is installed, staged theme drafts are submitted for approval and the linked approved workspace activates the draft; otherwise standalone publish remains the fallback.
- LayoutBuilder no longer imports AIOrchestrator classes. The LayoutBuilder AI capability now lives in `capell-app/ai-orchestrator` as an optional integration wrapper around LayoutBuilder Actions.
- `CapellFrontendThemePageAdapter` maps the active frontend page and loaded LayoutBuilder layout widgets into portable Theme Studio sections, including page-level widget assets.
- Capell package docs and the Capell app package catalogue have been updated for the new package names and bundle split.

```php
<?php

declare(strict_types=1);

namespace Capell\ThemeStudio\Core\Adapters;

use Capell\Frontend\Facades\Frontend;
use Capell\ThemeStudio\Core\Contracts\ThemePageAdapter;
use Capell\ThemeStudio\Core\Contracts\ThemeRuntimeSettings;
use Capell\ThemeStudio\Core\Data\HeroSectionData;
use Capell\ThemeStudio\Core\Data\ThemePageData;

class CapellFrontendThemePageAdapter implements ThemePageAdapter
{
    public function currentPage(): ThemePageData
    {
        $page = Frontend::page();

        return new ThemePageData(
            title: (string) $page->translation->title,
            brand: app(ThemeRuntimeSettings::class)->brandProfile(),
            sections: [
                HeroSectionData::from([

                    'body' => (string) $page->translation->content,
                ]),
            ],
        );
    }
}
```

The first adapter implementation maps content in this order: navigation from current site data when available, hero from page title and intro/content, shared section/content-block assets when the page exposes them, listing sections from page type metadata, CTA from page-level widget content, and footer from site footer data. When only a simple page exists, it returns a hero-only `ThemePageData` so runtime rendering is still truthful.

- [ ] **Step 4: Add run action**

```php
<?php

declare(strict_types=1);

namespace Capell\AIOrchestrator\Actions;

use Capell\AIOrchestrator\Data\AIOrchestratorRunData;
use Capell\AIOrchestrator\Support\AIOrchestratorModuleRegistry;
use InvalidArgumentException;
use Lorisleiva\Actions\Concerns\AsObject;

class RunAIOrchestratorCapabilityAction
{
    use AsObject;

    public function handle(string $capabilityKey, array $input = []): AIOrchestratorRunData
    {
        $capability = app(AIOrchestratorModuleRegistry::class)->findCapability($capabilityKey);

        throw_unless($capability instanceof \Capell\AIOrchestrator\Data\AIOrchestratorCapabilityData, InvalidArgumentException::class, 'AIOrchestrator capability is not registered: ' . $capabilityKey);

        $output = $capability->action::run($input);

        return new AIOrchestratorRunData($capabilityKey, $output);
    }
}
```
